# Customer Onboarding Guide

Status: Draft  
Audience: Pilot customers and early users  
Product: LeadGen Virtual Hub

## Before You Start

You need:

- A test or approved lead dataset.
- Permission to upload the data.
- A user account.
- Your assigned role.
- A goal for the session, such as identifying the top leads from a list.

## First Session Goal

By the end of the first session, you should have:

- Imported a lead dataset.
- Run enrichment.
- Reviewed enriched fields.
- Generated lead scores.
- Filtered a priority lead list.
- Exported or saved next actions.

## Step 1: Sign In

Sign in with the account provided by your workspace owner. If you cannot sign in, record the exact error message and contact support.

## Step 2: Import Leads

Upload the provided sample file or an approved customer dataset.

Check:

- Record count matches the expected number of rows.
- Required fields are mapped.
- Duplicate handling is clear.
- Invalid rows are shown with useful errors.

## Step 3: Run OSINT Enrichment

Start OSINT enrichment for the imported leads.

Review:

- Which fields were enriched.
- Which sources were used, if shown.
- Which records failed or returned no result.
- Whether any result looks stale or incorrect.

## Step 4: Run AI Enrichment

Start AI enrichment for the same lead set.

Review:

- Summaries or classifications are useful.
- Outputs are not overly confident where data is thin.
- Sensitive input data is handled according to policy.
- Failed enrichments are easy to understand.

## Step 5: Review Lead Scores

Open the scored lead list and compare high, medium, and low priority leads.

Ask:

- Do the top leads make sense?
- Can you understand why each score was assigned?
- Are there missing factors that should affect scoring?
- Are scores consistent when the same dataset is rerun?

## Step 6: Filter And Act

Filter by score, enrichment status, geography, industry, or other useful fields.

Save or export the lead list if export is enabled.

## Step 7: Invite A Teammate

If your role allows it, invite another user and assign the correct role. Confirm they cannot access features outside that role.

## What To Report During Pilot

- Anything that blocks progress.
- Incorrect or confusing enrichment.
- Lead scores that do not make sense.
- Slow pages or jobs.
- Missing explanations.
- Error messages that are unclear.
- Anything that would stop you from trusting the product.
